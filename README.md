# marx-paper-skills

马克思主义理论一级学科论文全流程 skill 系列。

## 适用学科

马克思主义理论一级学科（0305），含六个二级方向：

| 编号 | 二级方向 |
|------|----------|
| 1 | 马克思主义基本原理 |
| 2 | 马克思主义中国化研究 |
| 3 | 思想政治教育 |
| 4 | 国外马克思主义研究 |
| 5 | 马克思主义发展史 |
| 6 | 中国近现代史基本问题研究 |

适用于这六个方向下纯文科、无实证、纯文字性的课程论文、期末论文、学位论文。

## 设计理念

把马理论科论文写作拆成若干独立环节，每个环节做成一个 skill。各 skill 互相独立又彼此衔接，按需调用。

```
marx-paper（总入口路由，先说清系列默认设定，可选本次任务调整不改 skill）→ 选题（topic）→ 找文献 →（可选：阅读报告 read）→ 定义结构（structure）→ 整理素材（research）→ 写正文（quickwrite 快速式 / cowrite 引导式，二选一）→ 审核（review）→ 格式规范（format）
```

## marx-paper-format 详解

把马理论科论文草稿（markdown 或 docx）加工成格式规范、可直接提交的 Word 文档。

### 核心能力

- **脚注**：页下注，圈码 ①②③，每页重新编号，标注具体页码（开头问用户选 A 全自动还是 B 简单编号）
- **参考文献**：GB/T 7714-2015 规范，只列被引用过的，不标页码，不重复
- **Word 排版**：标题黑体居中、正文宋体小四、1.5 倍行距、首行缩进、参考文献单起一页、全篇黑色
- **一键 pipeline**：build_docx.py 一条命令跑完 pandoc→render→圈码→每页重编号，避免跳步
- **引号配对**：修正 pandoc 中文引号配对错误，直引号正确配对为弯引号
- **页码核实**：三种来源——有源 PDF 定位真实页码、无 PDF 标注待确认、来自 research 整理素材的复用原话直接用别人标好的出处（不必再核实，但完成后于对话里告知用户哪些是复用的），绝不编造

### 技术方案

| 难题 | 方案 | 脚本 |
|------|------|------|
| 圈码 ①②③ 不渲染 | customMarkFollows 属性 + 写入 Unicode 圈码字符 | footnote_circle_marks.py |
| 每页重新编号 | Word COM 查真实页码 + 按 w:id 重编号（解决重复引用覆盖问题） | renumber_footnotes_per_page.py |
| 中文引号配对 | 关 pandoc smart，自配对（奇数左、偶数右） | render_docx.py |
| 字体/行距/排版 | pandoc 转换 + python-docx 后处理 | render_docx.py |
| 跳步问题 | 一键 pipeline 串起四步 | build_docx.py |
| 单页超 20 脚注 | 降级成 (21)(22)... 并提示用户 | renumber_footnotes_per_page.py |

### 依赖

- [pandoc](https://pandoc.org/) — markdown ↔ docx 转换
- [python-docx](https://python-docx.org/) — docx XML 操作
- [pywin32](https://pypi.org/project/pywin32/) — Word/WPS COM 接口（查真实页码）
- Microsoft Word 或 WPS Office（任一即可）

## marx-paper-topic 详解

帮马理论科论文拟定、打磨、评估题目。三层已完成：特征识别 + 情况确认与分流 + 拟题配方。

### 已完成

- **第一步 特征识别**：5 种标题骨架、6 类研究路数、用词偏好三层、7 条深层气质
- **第二步 情况确认与分流**：三问（场景/有无资料/方向明确度）→ 场景细化 → 有资料/没资料路径分叉 → 没资料路径五个提问锚点
- **第三步 拟题配方**：方向 × 路数 × 骨架 × 切口 = 候选题目，筛选后留 10 个左右让用户选
- **衔接段**：主题确定后提醒用户去知网找 25-30 篇文献（关键词组合搜），征得同意建参考论文文件夹，衔接 research skill
- **44 题样例库**：按路数分类归档，每题标注骨架与切口（`marx-paper-topic/references/title-samples.md`）

## marx-paper-research 详解

把用户找来的参考文献里的引用整理成分类素材库。当前是雏形版本。

### 核心能力

- **读文献**：PDF（用 pdf skill）或 MD（直接读），建议转 MD（推荐 MinerU 软件，不建议模型转耗 token），但不强求
- **整理直接引用**：带双引号的原文（包括讲话、著作、文章里的话），按来源分文件夹——领导人、马恩、列宁、毛泽东、国外学者、政策文件等（具体分类跟用户商量确认）
- **整理间接引用**：无双引号但标了脚注的转述，不全整理——agent 自己对照结构判断有用与否（不推给用户），有用的按结构各部分分文件夹，每篇参考论文最多取 1 条
- **整理数据素材**：统计报告、调研数据、年度报告等，连同来源一起提取
- **提取完整出处信息**：提取脚注指向的完整文献信息（主要责任者、文献题名、出版社、出版年、页码），不是只标脚注编号——要到能直接写成 GB/T 7714 脚注的程度
- **素材库位置**：建在工作区根目录，与参考文献文件夹同级，不放进参考文献里
- **建文件夹前征得同意**：不擅自创建目录
- **垃圾站规则**：要删东西先扔工作区根目录的垃圾站，用户明确要求才真删

### 设计要点

- 复用别人用过的引用是合理做法：别人已标好出处，直接拿来用，不必每条都翻原书核实
- 直接引用按来源分、间接引用按结构分——两种引用组织方式不同
- 间接引用每篇最多 1 条，保证素材来源分散
- 衔接：直接引用直接用（人物原话要溯源到原始出处）、间接引用换说法用（少复用层层转述）；format 编排脚注参考文献，复用的不必再核实但告知用户

## marx-paper-structure 详解

把题目变成完整的结构方案（大标题、小标题、各部分字数）。在选题之后、整理素材之前运行。当前是雏形版本。

### 核心能力

- **工作方式**：先给一版工整排比式结构（最常见），不满意再给两三个其他思路，再不满意接着调
- **默认 scaffold**：约11500-12000 字，引言 350-600、主体 3 大标题（每个含引子段100-250 + 3 小标题各800-1400中心约1100）、结语约500（可选）
- **每个大标题下有引子段**：100-250字，承上启下点明主旨，再拆小标题展开（样本普遍规律）
- **结构规律来自顶刊样本**：《马克思主义研究》25-26 年 200+ 篇统计，3 大标题主流（54%）、4 大（31%）、5 大（11%）；约 73% 无结语
- **大标题起法四种**：递进式排比（最常见，默认先用）、概念-展开式（冒号）、从X到Y、直接陈述
- **小标题起法两种**：冒号式拆解、从X到Y式
- **6 个方向各 5 个完整样本**存于 `references/structure-samples.md`（共30篇，每篇列全大小标题）

## marx-paper-quickwrite 详解

用结构做骨架、素材做血肉一气撜写整篇（快速式·纯器写作）。在结构定稿和素材整理完成之后运行。当前是雏形版本。需引导用户共写的场景见 marx-paper-cowrite（占位待实现）。

## marx-paper-cowrite 详解

引导式写作：和用户共写。占位，第二批单独实现。先定框架→引导重点读哪篇→提问题收思想→用素材再写。慢但更有灵魂、更贴合用户本意。

## marx-paper-read 详解

文献阅读报告（可选/跳过）。在找文献后、定结构前可选插一步。逐篇读+提炼，用三问骨架出报告，让用户不翻原文也能回想这篇讲了啥、怎么论证、对自己有什么用。也为 cowrite 引导式提供“先读哪篇”依据。

### 三问骨架

- **What**：作者的核心主张是什么（盯各级标题+每段主题句，拆出核心观点/概念/问题）
- **How**：作者如何论证（主要逻辑、研究路径、理论贡献；挂精读清单：有意思在哪、主流解释不足、相比可替代解释好在哪、引文/方法/发现/讨论各文本块）
- **So what**：对我有何启发（和已知知识/本选题的关联、可用之处、批判思考、是否重点文献）

### 研究类型识别（马理论科适配）

理论阐释型 / 文本解读型 / 历史考辨型 / 比较研究型 / 对策研究型 / 实证调研型（偶有）。不是社科实证那套“定量/质性/混合”。

### 学写作

报告含标题怎么起、各级标题怎么排、起承转合怎么搭，用户写自己论文能照搬结构手法。
