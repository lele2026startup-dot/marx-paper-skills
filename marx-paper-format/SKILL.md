# 马克思主义理论学科论文格式规范与脚注参考文献

## 这个 skill 做什么

把马理论科论文草稿（markdown 或 docx）加工成格式规范、可直接提交的 Word 文档。覆盖全流程格式：引用标注（脚注+参考文献）、页码核实、Word 排版（字体、字号、行距、缩进、分页、颜色）。

核心规则（这几条是国标，不可商量）：
- **脚注**：页下注，凡引用必标注，必须标具体页码
- **参考文献**：只列正文脚注中出现过的文献，不标页码，不重复列，未引用的不得出现

## 工作流

### 第〇步：圈码编号方式确认（开头先问）

开始格式化之前，先跟用户确认脚注编号方式，因为两种方式各有利弊，由用户选：

- **A 全自动圈码**：由脚本自动排好圈码 ①②③ 并每页重新编号。好处是开箱即用、不用手调；代价是依赖脚本和 Word COM，偶尔可能因模型能力或环境问题出错（如单页脚注超过 20 个时圈码会降级为 (21)(22)... 形式，脚本会提示）。
- **B 简单数字编号**：只用 pandoc 默认的 1,2,3... 从头到尾编号，不每页重新编号、不用圈码。好处是稳定不出错、符合模型舒适区；代价是用户需要在 Word 里手动改成圈码和每页重编号（手动调整不难、耗时少）。

**每次都问**，不要替用户默认。选 A 走完整脚本链（build_docx.py 默认模式），选 B 走简化链（build_docx.py --no-circle）。

### 第一步：摸清项目环境

打开 skill 后第一件事不是动手加脚注，而是先看清所处项目的全貌。这一点很重要——不同论文项目的目录结构、源文献位置都不一样，盲目动手会找不到 PDF 也搞错格式要求。

具体做这些：
- 扫描当前工作目录及其上层，找到论文草稿（markdown 或 docx）
- 找可用的源文献：原书 PDF、原刊 PDF、电子书等
- 找格式模板或撰写要求文档（如学校/学院的论文规范）
- 然后跟用户确认三件事：
  1. 源 PDF 在哪个目录？（有些项目可能根本没有原 PDF，这点要明确问）
  2. 这次正文行距多少？（毕业论文常为多倍行距 1.25，课程论文常为 1.5 倍行距——以用户本次作业要求为准）
  3. 有没有指定的字体/格式模板？

不要假设源 PDF 一定存在。没有 PDF 就如实告诉用户，走"页码待确认"流程，绝不能编。

### 第二步：通读草稿，定位所有需要脚注的位置

逐段通读全文，找出每一处：
- **直接引用**：引用了原文原话（哪怕几个字）
- **间接引用**：概括、转述了他人观点、理论、论断

这一步要尽量找全。GB/T 7714 的要求是"凡引用必标注"——漏标在学术规范上等同于学术不端。宁可多标，不可漏标。把每一处需要脚注的位置和它引用的是哪本文献记下来。

### 第三步：核实页码（最关键的一步）

这是最容易偷懒、也最不能偷懒的一步。脚注里的页码必须是引文在原文献中的真实页码。

- **有源 PDF**：用 pdf skill（pdfplumber/pypdf 提取文本，Poppler 渲染页面为图片做视觉核对）定位引文所在页，记录真实页码。
- **无源 PDF**：明确告诉用户"这条引用的页码我无法核实"，请用户提供，或在脚注里标注"[页码待核实]"占位**并在对话里逐条列出哪些脚注页码未核实，请用户提供**。严禁回填一个"看起来合理"的数字——回填数字 = 编造页码 = 学术诚信硬伤。用占位符 + 对话告知用户，优于填一个数。
- **来自整理素材（复用别人用过的原话）**：如果这条引用来自 marx-paper-research skill 整理出来的原话素材——也就是别人论文里已经引用过、并且已经标注了出处和页码的原话——那这些标注信息可以直接复用，不必再去原 PDF 核实，因为别人已经核实过了。这是常见做法：复用别人用过的、已经标好出处的原话，等于站在前人肩膀上。但有一条要求：文档格式化完成后，要在**对话里**（不是写在论文里）跟用户说明，哪些脚注是复用整理素材里的原话，让用户心里有数——万一要检查或追溯，用户可以再去核实。
- **绝不能凭感觉编页码。** 这是学术诚信的底线。前三种情况都有真实信息来源（PDF、用户、整理素材），唯独"凭感觉"没有任何来源，这条红线不能碰。

为什么必须读 PDF 而不是 markdown：同一文献的 markdown 电子版和 PDF 版页码往往不一致——markdown 没有固定的"页"概念，分页随渲染方式变化。PDF 才是引文页码的权威依据。

### 第四步：编写脚注

按 GB/T 7714-2015 格式编写每条脚注。完整格式见 `references/gbt7714-format.md`，这里列最常用的：

- **专著**：`主要责任者.文献题名[M].出版地：出版者，出版年.起止页码.`
  - 例：`卢卡奇. 历史与阶级意识[M]. 北京: 商务印书馆, 1999: 143-149.`
- **期刊**：`主要责任者.文献题名[J].刊名，年，卷(期)：起止页码.`
- **译著**：`原著者.文献题名[M].译者，译.出版地：出版者，出版年.起止页码.`

几个易错点（务必注意）：
- 出版地后用**全角冒号**"："，不是英文冒号
- 页码直接写数字范围，**不加"第"和"页"字**（如 `143-149`，不是"第143-149页"）
- **重复引用**：连续引用同一文献用"同上（书），第╳页"（注意这里反而保留"第"和"页"）；非连续引用只注著者、文献名、页码
- 编著/译著在编者、译者姓名后加"编""译"

### 第五步：生成参考文献表

在文末生成参考文献，规则严格：
- **只收录**正文脚注中出现过的文献
- **不标页码**
- **不重复**（同一文献多处引用，参考文献表里只列一次）
- 按文中**首次引用顺序**编号 [1][2][3]...
- **未引用的文献不得出现**——这条是硬性规定。把只是"思想上参考过"但正文没有实际引用的文献列进去，属于违规
- **生成后做去重校验**：对著录逐条比对（归一化责任者+题名+版本后比对），同一文献多处出现必须合并为一条。常见错误：同一本书不同译本/同一年同版被列为两条（如《逃避自由》刘林海译本出现两处[1]和[5]）。人眼容易漏，务必过一遍。
- **期刊论文著录要从知网取**：如某条来自期刊论文且著录不全（只有题名无刊名/年/卷期/页码），让用户去知网该论文页面点“引用”图标复制完整著录回来填，不要凭空凑。凭凑的刊名/卷期/页码几乎必错。
- **经典原著用最新最全版本**：同一部经典如果有更新/更全的译本或版本，优先用最新版本的页码。复用 marx-paper-research 整理出的原话出处时，若原出处是老版本，应查或提醒用户换成最新版本页码（现阶段是原则要求，后续会做独立的原著页码检索库 skill 集中确认）。

为什么这条这么重要：GB/T 7714 和各高校规范都明确写"未引用的文献不得出现"。参考文献表是"被引用文献的完整列表"，不是"我读过的书单"。这个区别要刻在脑子里。

### 第六步：渲染成 Word

用一键脚本 `scripts/build_docx.py` 把 markdown 转成规范 docx。它把 pandoc 转换、字体排版、圈码化、每页重编号四步串成一条命令，从机制上避免漏跑或跳步。

```bash
# A 全自动圈码（默认）
python scripts/build_docx.py 草稿.md 成品.docx

# B 简单数字编号（用户手动调圈码）
python scripts/build_docx.py 草稿.md 成品.docx --no-circle
```

脚本内部依次执行：
1. `pandoc --from=markdown-smart --to=docx`（关掉 smart 避免中文引号配对错误）
2. `render_docx.py` 设置字体/行距/缩进/排版
3. `footnote_circle_marks.py` 圈码化（仅 A 模式）
4. `renumber_footnotes_per_page.py` 每页重编号（仅 A 模式）

为什么用一键脚本而不是分步跑：分步手动跑时容易自作主张跳过某步（比如信任 render_docx 自带功能而不跑 renumber），导致圈码/每页重编号失败。一键脚本从机制上杜绝跳步。

**markdown 写作时的注意事项**（避免渲染问题）：
- 引用必须用 pandoc 脚注语法 `[^n]`，绝不能用行内方括号 `[n]`（pandoc 不认，会渲染成莫名数字）
- 不要用 `---` 做视觉分隔（pandoc 会渲染成横线）
- 标题块（frontmatter 的 title + 摘要 + 关键词）要保护好，编辑正文时不能覆盖掉
- 正文段落前后空行要规范，避免 pandoc 误判段落为标题层级

### 第七步：自检

交付前逐项检查：
- [ ] 脚注数量与正文引用标记一致
- [ ] 参考文献每条都能在脚注中找到对应（无多余、无遗漏）
- [ ] **参考文献不重复**（同一文献只列一次；去重校验过）
- [ ] **参考文献每条著录完整**（专著有出版地/社/年、期刊有刊名/年/卷期；残缺的已让用户从知网补齐）
- [ ] 所有页码已核实，无"[页码待核实]"残留（或已逐条告知用户哪些待核实、请用户补）
- [ ] **同一文献多引用的页码未被误复用为同一条页码**
- [ ] 经典原著使用了最新/最全版本（或已提醒用户换版本）
- [ ] 脚注格式：圈码、每页重编号、全角冒号、页码无"第/页"字
- [ ] 字体/行距/缩进正确
- [ ] 打开 docx 目视检查一遍脚注是否真的出现在页面底部

## 修补已有论文

当输入是已经写好的论文（可能有也可能没有脚注）时，工作流调整为：
1. **扫描现有脚注**：逐条核实页码和格式是否正确，错误的改正
2. **找漏标**：找出"该加脚注但没加"的位置，补全
3. **重建参考文献表**：基于所有脚注重新生成，确保配套
4. **重新渲染**：用脚本输出格式正确的 docx

## 已知技术事项（迭代时关注）

**圈码脚注 ①②③——已解决**：用 `customMarkFollows="true"` 方案。在正文每个 `w:footnoteReference` 上加该属性，并在 run 里写入圈码字符 `<w:t>①</w:t>`，Word 就显示我们指定的字符而非自动编号。底部脚注的 `w:footnoteRef` 同步替换成圈码字符（不加 FootnoteReference 样式，避免上标变小）。`w:id` 关联保留，仍是真脚注（删正文标记，底部脚注跟随消失）。由 `scripts/footnote_circle_marks.py` 实现。

**每页重新编号——已解决**：用 Word/WPS 的 COM 接口（pywin32）打开 docx，调用 `Footnote.Reference.Information(wdActiveEndPageNumber)` 获取每个脚注引用在 Word 里的真实页码，按页分组后每页内按出现顺序重新赋 ①②③。**用 footnote 的 w:id 作重编号映射的 key**（不用旧圈码字符）——因为同一文献被重复引用时，footnote_circle_marks 会把多个引用点写成同一个圈码字符，用圈码字符作 key 会导致映射被反复覆盖、每页重编号错位。w:id 是每个引用点唯一的，不会冲突。由 `scripts/renumber_footnotes_per_page.py` 实现。**重要：必须用 Word COM 而非 LibreOffice 转 PDF**——LibreOffice 与 Word 的分页算法不同，用 LibreOffice 的分页去编号会导致 Word 打开时编号错位。依赖 pywin32 + Word 或 WPS（任一即可，COM 接口通用）。编号是渲染时算死的，不是实时自动的——用户在 Word 里手动新增脚注后需重跑脚本。这与本项目工作流（markdown 改 → 脚本渲染 → 交最终版）匹配。

**单页脚注超过 20 个——降级处理**：Unicode 圈码 ①-⑳ 只到 20，22-35 的扩展圈码字符（22-35）常见字体（宋体等）多无字形，显示不出来。脚本对超过 20 的自动降级成 (21)(22)... 形式，并在结束时提示用户该页脚注过多。每页重编号正确时，单页超过 20 个脚注极少见，这是兑底处理。

**pandoc 脚注 ID 偏移**：pandoc 从 markdown 的 YAML frontmatter 会预留一些内部脚注 ID，导致正文脚注的 XML ID 不是从 1 开始（如从 9 开始）。这不影响圈码显示（圈码是我们按出现顺序算的，与 XML ID 无关），但开发调试时要心里有数。

**中文引号配对——已解决**：pandoc 的 smart 扩展在中文环境里引号配对不可靠，会把成对引号全转成右引号 “”。解决方案：pandoc 命令用 `--from=markdown-smart` 关掉智能引号，保留直引号 \"，由 render_docx.py 的 `fix_quotes_in_paragraph` 按 "奇数位左引号、偶数位右引号" 规则正确配对。含脚注引用的段落逐 run 独立配对（避免跨 run 错位）；纯文本段落全局配对。

## 相关文件

- `references/gbt7714-format.md` — 各类文献（专著、期刊、学位论文、会议论文、网络资料、报纸等）的详细著录格式与示例，编写脚注时查阅
- `scripts/build_docx.py` — 一键 pipeline：pandoc→render→圈码→每页重编号，一条命令跑完（--no-circle 可跳过圈码步骤）
- `scripts/render_docx.py` — docx 后处理脚本，负责字体、行距、脚注编号格式、引号配对
- `scripts/footnote_circle_marks.py` — 把脚注引用和底部标记改成圈码 ①②③（customMarkFollows 方案）
- `scripts/renumber_footnotes_per_page.py` — 让圈码脚注每页重新编号（Word COM 查真实页码 + 按 w:id 重编号，解决重复引用覆盖问题）
