# 马克思主义理论学科论文审核

## 这个 skill 解决什么问题

写完正文后，自己审自己容易护短——写的人看不出自己的问题。这个 skill 用独立 agent（新上下文）系统过一遍正文，检查是否符合系列各 skill 的要求，列出问题和修改建议。

它在 marx-paper-write 写完正文后、marx-paper-format 排版前运行。审核通过再排版，避免排版后发现内容问题又返工。

## 审核清单

逐项检查，每项标注"通过/问题/建议"：

### 引用与溯源
- [ ] **领导人/经典作家原话溯源**：领导人、马克思恩格斯、列宁、毛泽东等的原话，脚注是否溯源到本人原始出处（讲话/著作），而不是标"转引自某论文"
- [ ] **数据/统计/调研溯源**：统计数据、调研数据是否溯源到最源头出处（报告本身），不是标转引论文
- [ ] **学者观点指名**：引用学者观点是否写出具体姓名，没有"某位学者说""有学者认为"这类泛指
- [ ] **被引学者知名度**：被引学者是否为知名学者或高被引学者，没有引用小作者的细节观点
- [ ] **直接引用选取**：直接引用的是否为根本性话语，不是细节琐碎表述

### 结构与字数
- [ ] **引言字数**：引言是否在 350-600 字范围内（structure 定的，不是建议）
- [ ] **结语字数**：结语是否约 500 字（如有的话）
- [ ] **各部分字数**：主体各部分是否按 structure 方案的字数分配，没有严重超长或过短
- [ ] **每部分有引用**：3×3=9 个部分，每个部分是否都有引用/脚注支撑，没有"裸写"的段落

### 语体与风格
- [ ] **破折号克制**：破折号是否克制（全文不超过三五处），没有 AI 式滥用
- [ ] **反问句克制**：反问句是否克制，引子段没有"X何以…？Y为何…？"设问式
- [ ] **学科语体**：是否用马理论科学术语体，没有时评/公文/网络词汇（如"国运""国本"等）
- [ ] **起始性语言**：是否有重复常识性、纲领性表述而缺乏论证的"起始性语言"（应改成说理性语言：抛观点→表明立场→论证）

### 论证质量
- [ ] **论-据-证闭环**：每个段落是否有"论-据-证"闭环，不只"观点+例子"，有论证分析
- [ ] **不用别人观点证明别人观点**：没有"用马克思观点反对别人观点"这种缺"证"的闭环
- [ ] **概念界定**：核心概念首次使用是否有清晰定义，全文概念同一性是否保持

### 格式预检（format 前的 sanity check）
- [ ] **脚注语法**：引用是否都用 pandoc 脚注语法 `[^n]`，没有行内方括号 `[n]`
- [ ] **无 `---` 分隔符**：正文没有用 `---` 做视觉分隔（会被 pandoc 渲染成横线）
- [ ] **标题块完整**：frontmatter title + 摘要 + 关键词是否齐全，没被覆盖
- [ ] **段落空行规范**：正文段落前后空行是否规范，避免 pandoc 误判为标题

## 工作方式

1. **通读全文**：把正文从头到尾读一遍，不是抽样
2. **逐项过清单**：按上面清单逐项检查，每项标"通过/问题/建议"
3. **输出审核报告**：把问题按严重程度分类（硬伤/建议），给具体修改建议（指明哪段、怎么改）
4. **不要自己改**：审核 agent 只挑问题、给建议，不直接改正文（改是 write agent 的事，保持职责分离）
5. **硬伤优先**：溯源问题、每部分无引用、字数严重超标这类硬伤优先列；破折号多一处这种小问题归到"建议"

## 为什么独立 skill 而不是合进 write

- **单一职责**：跟系列其他 skill 一致
- **新上下文更客观**：同一个 agent 写完自己审，容易护短（自己写的东西自己看不出问题）。独立 agent 没有写作时的思维定势，更客观
- **系统检查**：把分散在各 skill 的要求集中成一份清单，系统过一遍，不会漏
- **流程清晰**：用户写完单独调用，审核通过再排版

## 衔接

审核完后：
- 有硬伤 → 回到 marx-paper-write 按建议改正文，改完再审一遍
- 只有建议、无硬伤 → 用户决定改不改，然后衔接 marx-paper-format 排版

## 备注

- 当前是雏形版本，审核清单根据实际使用磨合完善
- 审核清单的各项要求来自系列其他 skill（write 的写作规范、structure 的字数、research 的溯源规则等），本 skill 是把这些要求的检查集中起来
