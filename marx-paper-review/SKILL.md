# 马克思主义理论学科论文审核

## 这个 skill 解决什么问题

写完正文后，自己审自己容易护短——写的人看不出自己的问题。这个 skill 用独立 agent（新上下文）系统过一遍正文，检查是否符合系列各 skill 的要求，列出问题和修改建议。

它在 marx-paper-quickwrite（快速式）或 marx-paper-cowrite（引导式）写完正文后、marx-paper-format 排版前运行。审核通过再排版，避免排版后发现内容问题又返工。

## 审核清单

逐项检查，每项标注"通过/问题/建议"：

### 引用与溯源
- [ ] **领导人/经典作家原话溯源**：领导人、马克思恩格斯、列宁、毛泽东等的原话，脚注是否溯源到本人原始出处（讲话/著作），而不是标"转引自某论文"
- [ ] **数据/统计/调研溯源**：统计数据、调研数据是否溯源到最源头出处（报告本身），不是标转引论文
- [ ] **学者观点指名（硬伤）**：引用学者观点是否写出具体姓名，没有"一位学者""有学者认为""某位学者说""一些学者也"这类泛指——出现即硬伤，必须改成具体姓名或改为不点名综述
- [ ] **点名必可溯**：正文中点名的学者，参考文献里必须有可核实条目。提了人名却参考文里查不到、是硬伤（如"以李某某为代表"但参考文献无该人）
- [ ] **被引学者知名度**：被引学者是否为高被引或知名学者（间接引用二手论文核心观点尤其须高被引/知名）；没有引用无法核实/小众作者的细节观点
- [ ] **直接引用选取**：直接引用的是否为根本性话语，不是细节琐碎表述
- [ ] **引号=原文（硬伤）**：双引号里的内容是否与原文逐字一致，有无在引号内增删改字、或把转述/改写放进引号当成 verbatim 直接引用
- [ ] **同文献多引用页码是否重复**：同一本书被多次引用，各脚注页码是否按各引文实际所在页逐一核实，有无复制粘贴忘改、多条引用都用同一条页码
- [ ] **页码可核实性**：是否有"无法核实却填了一个数"的页码；无源 PDF 的页码是否标占位+告知用户，没有偷偷填编造页码

### 结构与字数
- [ ] **引言字数**：引言是否在 350-600 字范围内（structure 定的，不是建议）
- [ ] **结语字数**：结语是否约 500 字（如有的话）
- [ ] **各部分字数**：主体各部分是否按 structure 方案的字数分配，没有严重超长或过短
- [ ] **每部分有支撑（硬伤）**：3×3=9 部分（或课程论文各部分）是否每部分都有依据支撑（引用/脚注/史实/数据），有无整部分裸写。贵精不贵多、不为凑数硬凅 OK，但整部分无支撑是硬伤

### 语体与风格
- [ ] **破折号克制**：破折号是否克制（全文不超过三五处），没有 AI 式滥用
- [ ] **反问句克制**：反问句是否克制，引子段没有"X何以…？Y为何…？"设问式
- [ ] **AI 典型句式克制**："不是…而是…""不仅…更是…""并非…而是…""而非…"这四式是否节制（一篇 12000 字合计不超过三五处），能换陈述句的是否换了
- [ ] **散文腔/文学化（硬伤）**：是否被研究资料带偏、模仿研究对象的文风。研究鲁迅不能写成鲁迅腔，研究文学不能用诗化散文。格言式/抒情式收尾（"未及名言""戳到痛处""最朴素也最扎心的一句提醒"）出现即硬伤
- [ ] **口语化（硬伤）**：是否有口语化表达（"关心一件事儿""萦绕的痛气相通"等），口语化即硬伤
- [ ] **学科语体**：是否用马理论科学术语体，没有时评/公文/网络词汇（如"国运""国本"等）
- [ ] **起始性语言**：是否有重复常识性、纲领性表述而缺乏论证的"起始性语言"（应改成说理性语言：抛观点→表明立场→论证）

### 论证质量
- [ ] **论-据-证闭环**：每个段落是否有"论-据-证"闭环，不只"观点+例子"，有论证分析
- [ ] **不用别人观点证明别人观点**：没有"用马克思观点反对别人观点"这种缺"证"的闭环
- [ ] **概念界定**：核心概念首次使用是否有清晰定义，全文概念同一性是否保持

### 格式预检（format 前的 sanity check）
- [ ] **脚注语法**：引用是否都用 pandoc 脚注语法 `[^n]`，没有行内方括号 `[n]`
- [ ] **无 `---` 分隔符**：正文没有用 `---` 做视觉分隔（会被 pandoc 渲染成横线）
- [ ] **标题块完整**：frontmatter title + 摘要 + 关键词是否齐全，没被覆盖
- [ ] **段落空行规范**：正文段落前后空行是否规范，避免 pandoc 误判为标题
- [ ] **参考文献完整性**：参考文献每条是否完整（专著有出版地/出版社/年、期刊有刊名/年/卷期/页码），有无只有题名无期刊信息。期刊论文的著录最好提醒用户去知网点"引用"图标复制标准著录
- [ ] **参考文献不重复**：同一文献是否多处重复列（如同名同版本书出现两次）；应去重，同一文献只列一次
- [ ] **参考文献与脚注对齐**：参考文献表是否只列被脚注引用过的（无未引文献）、无遗漏，点名的学者都有对应条目

## 工作方式

1. **通读全文**：把正文从头到尾读一遍，不是抽样
2. **逐项过清单，不许跳过**：按上面清单**逐项检查，每项都写出"通过/问题/建议"，通过的也要写出"通过"，不许不对不在报告里不提**。这是反偏規、防偏規的机制。
3. **输出审核报告**：把问题按严重程度分类（硬伤/建议），给具体修改建议（指明哪段、怎么改）
4. **不要自己改**：审核 agent 只挑问题、给建议，不直接改正文（改是 quickwrite/cowrite 写作 agent 的事，保持职责分离）
5. **硬伤优先**：溯源问题、指名泛指、引号非 verbatim、每部分裸写、页码编造、参考文献残缺/重复/漏点名这类硬伤优先列；破折号多一处这种小问题归到"建议"
6. **反偷懒机制**：如果报告只报了破折号/反问句这类最浅层的项，而没覆盖引用真实性（引号逐字、页码重复/可核实性）、参考文献完整性/重复、指名、散文腔/口语化、AI 句式等项——视为走过场，必须重审、补全项。不许"只报最明显的两条了事"。

## 为什么独立 skill 而不是合进 write

- **单一职责**：跟系列其他 skill 一致
- **新上下文更客观**：同一个 agent 写完自己审，容易护短（自己写的东西自己看不出问题）。独立 agent 没有写作时的思维定势，更客观
- **系统检查**：把分散在各 skill 的要求集中成一份清单，系统过一遍，不会漏
- **流程清晰**：用户写完单独调用，审核通过再排版

## 衔接

审核完后：
- 有硬伤 → 回到写正文的 skill（marx-paper-quickwrite 或 marx-paper-cowrite）按建议改正文，改完再审一遍
- 只有建议、无硬伤 → 用户决定改不改，然后衔接 marx-paper-format 排版

## 备注

- 当前是雏形版本，审核清单根据实际使用磨合完善
- 审核清单的各项要求来自系列其他 skill（quickwrite/cowrite 的写作规范、structure 的字数、research 的溯源规则等），本 skill 是把这些要求的检查集中起来
