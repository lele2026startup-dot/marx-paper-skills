# 马理论科论文·文献阅读报告

## 这个 skill 解决什么问题

选题定了、文献找来了（通常 25-30 篇，由 marx-paper-topic skill 提醒用户去找），下一步可以**可选**插一步：帮用户读懂这批文献、给每篇出一份阅读报告。

为什么要有这步：用户干读 25-30 篇原文很累，容易读完就忘、抓不住重点。这 skill 先逐篇读+提炼，给用户三样东西——

1. **一篇文献的骨架**：它主张什么、怎么论证、什么类型，让用户不用重读原文也能回忆起来
2. **写作可学的点**：它标题怎么起的、各部分怎么排、怎么衔接——用户自己要写论文，学别人怎么组织
3. **对自己的启发**：这篇和自己的选题有什么关联、能借什么、哪里不能照搬

这一步是**认知打底**，不是抠素材。抠引用、抠出处是后面 marx-paper-research skill 的事——那个在定结构之后做，目的是把别人用过的话和出处提取出来备用。本 skill 是在 research 之前，目的是让用户**读懂**。两者读的是同一批文献，但目的不同、先后不同：先 read 懂，后 research 抠。

**可选可跳过**：急着写可跳过本步直接进 marx-paper-structure。但走了这步，后面定结构、写正文会更有底，尤其走 marx-paper-cowrite 引导式写作时，本 skill 的报告能告诉你"重点读哪篇、跟用户聊哪篇"。

## 三问骨架（阅读报告的主干）

每篇文献的阅读报告用**三个核心问题**作主干，从 What 到 How 到 So what，由浅入深：

### 一、What —— 作者的核心主张是什么

回答"这篇到底在说什么"。怎么看：

- **盯各级标题**：大标题、各节小标题连起来读，往往就是作者的论证骨架
- **盯每段的主题句**：马理论科论文段落多"总—分"结构，段首那句常是该段论点
- 把标题链 + 主题句串起来，就是作者的核心框架

报告里这一问产出：**核心观点**（一两句话能说清的那种，不是复述全文）、**核心概念**（作者反复用、当作分析工具的几个关键概念）、**核心问题**（作者要回答的那个问题）。

### 二、How —— 作者如何论证

回答"凭什么这么说"。这是精读的主体，看作者用什么把主张立住。马理论科文献的论证跟社科实证论文不一样（没有问卷/访谈/数据），要看的精读清单见下面"精读清单"一节。

报告里这一问产出：**主要论证逻辑**（作者分几步、用什么把论点撑起来）、**研究方法/路径**（文本解读？历史考辨？比较研究？）、**理论贡献**（验证/扩展/修正了既有理论，还是提出新概念、建构新框架、整合既有说法）。

### 三、So what —— 对我有何启发

回答"读了对我有什么用"。这一问把文献和**用户自己**接起来：

- 和我已知的知识有什么关联：印证了什么、推翻了我什么旧印象、补了我什么空白
- 和**我的选题**有什么关联：能借它的什么（论点、概念、论证角度、写作结构），哪里不能照搬（学科语境不同、论域不同）
- 这篇值不值得重点读、回头写正文时重点复用

报告里这一问产出：**批判性思考**（我的判断、保留意见）、**对本选题的可用之处**、**是否重点文献**（给 cowrite 引导式标重点用）。

## 精读清单（挂在 How 之下的具体看点）

"How 作者如何论证"这一问，展开成下面这份精读清单。不是每篇都面面俱到——按这篇文献的实际情况挑相关的看。清单分两块：**通用看点**（每篇都适用）+ **模块精读**（按论文结构分块看）。

### 通用看点

- **有意思的地方在哪**：这篇哪点让你觉得"诶，这个点好"。可能是戳中了一个常识或既有解释答不了的难题，可能是对一个老问题给了新解，可能是把两个本来不相干的概念接上了。先把"有意思"指出来，再谈"好在哪"。
- **是否已有主流解释**：针对作者要回答的那个问题，学界有没有主流说法？主流说法的不足在哪？作者是不是冲着这个不足去的。
- **相比可替代解释，作者的分析好在哪里**：同一个问题别人怎么说，作者怎么说，作者的说法比别的强在哪。
- **理论贡献往哪归**（择一或几）：验证既有理论 / 扩展既有理论 / 修正既有理论 / 提出新概念 / 建构新框架 / 整合零散说法 / 完善某概念的操作化。马理论科文献里"建构新框架""整合零散说法""对经典文本给出新解读"这几类很常见。
- **学写作**（对用户写自己论文极有价值，别跳过）：
  - 标题怎么起的：主标题+副标题的关系、用词风格
  - 各级标题怎么排：为什么这节在前、那节在后、各部分怎么衔接——把标题结构跟作者的论证逻辑对上
  - 整篇的起承转合怎么搭

### 模块精读（按论文结构分块）

一篇论文大致分几块，逐块看作者怎么写、能学什么：

**引言 / 文献综述**
- 作者依托的**核心理论 / 核心文本**在哪（理论起点）——马理论科论文常以某部经典、某段论述、某个理论框架为起点，把它指出来
- 作者对现有研究什么判断：哪些做过了、哪些还不够
- 文献综述怎么**避免罗列感**：是批判性地提取出研究问题，而不是"张三说了什么、李四说了什么"地排排坐。看作者怎么把现有文献的不足**铺垫**出来，为自己的问题腾地方
- 怎么点明本研究的**理论意义和现实意义**

**研究方法 / 研究路径**
- 作者用了什么方法或路径回答自己的问题
- 为什么这条路适用于这个问题（方法跟问题的匹配）
- 马理论科文献多数不是实证，路径常是：经典文本解读、历史文献考辨、概念考辨、比较研究、理论分析、政策文本分析。如实标是哪一种
- 偶有实证（思想政治教育方向偶有问卷、访谈、案例调研）：如实标，并看作者怎么把经验材料变成论证证据、而不是孤立的数字或案例

**研究发现 / 主体论证**
- 这是马理论科论文的"肉"，往往不是"发现"而是"论证"：作者分几步把核心论点立起来
- 每一步论证如何回应开头的研究问题
- 论证是否支撑作者的理论框架、是否挑战了既有解释
- 围绕理论和研究问题，让材料（经典文本原句、史料、案例）成为论证的证据，而非堆摆

**讨论与结论**
- 作者如何超越对结果的简单重复、体现出理论价值
- 指明的未来研究方向

## 研究类型识别

报告里给这篇标个类型，方便用户一眼分类。马理论科文献的类型不是"定量/质性/混合"那套（那套是社科实证论文的），按下面分：

- **理论阐释型**：阐释某个经典论断、某段论述、某个概念的含义，最常见
- **文本解读型**：对某部经典、某篇著作做细读、训诂、义理分析
- **历史考辨型**：对某个历史事件、某段思想史做史料梳理与辨正
- **比较研究型**：中西比较、古今比较、不同流派比较、马中外比较
- **对策研究型**：针对某个现实问题提对策思路（中国化方向多见）
- **实证调研型**：问卷、访谈、案例调研（思想政治教育方向偶有，少见）

具体往下还可以细分（如文本解读型里分"逐句训诂"和"整体义理"），但不必太细，能帮用户归类即可。多数马理论科文献落在前四类。

## 产出去哪：论文工作区里的“阅读思考/”文件夹

本 skill 的产出都放在论文工作区根目录下的 **“阅读思考/”** 文件夹（与后续 marx-paper-research 建、“素材库/”、“参考文献/”、“论文结构/”、“论文正文/”同级）。这个文件夹是本 skill 专属产出地，且本质上是**后续写作的素材源之一**——阅读报告 + 用户读后的想法，加起来是后面定结构、就手冸 pro cowrite 引导式写作时的素材。

文件夹结构：

```
工作区/
├── 阅读思考/                          ← 本 skill 建（建前征用户同意）
│   ├── 报告/                          ← 每篇文献一份阅读报告
│   │   ├── 阅读报告·吴晓明().md
│   │   └── ...
│   ├── 用户想法/                      ← 用户读报告后冒的想法、agent 代笔落盘
│   │   ├── 想法_随着文献名().md
│   │   └── ...
│   └── 重点文献小结.md                ← 一页纸点题，给 cowrite 看先读哪篇
├── 参考文献/                          ← marx-paper-topic 找来的文献
├── 素材库/                            ← 下游 marx-paper-research 建
├── 论文结构/                          ← marx-paper-structure 建
└── 论文正文/                          ← 写作 skill 建
```

两边区别：本 skill 的“阅读思考/”是**认知性**保质（用户读懂了什么、对这个选题怎么想的）；下游 research 的“素材库/”是**素材性**保质（引用原文 + 出处，准备进脚注）。两者都服务于后续写作，但一个装的是“用户头脑里的东西”，一个装的是“文本里的原话和出处”，不要混。

## 工作流程

### 第一步：跟用户确认读的进度安排

用户找来的文献（通常 25-30 篇）**本来就要全读**——你找文献就是为了读它，不存在“只读几篇”的问题。这一步要跟用户确认的不是“读哪几篇”，而是**读的进度安排**：

- **逐篇读、逐篇出报告**：所有文献一篇出一份完整阅读报告。用户一次给多少篇就处理多少篇，读一篇出一篇，不必等整批齐才动工。这对用户最完整，也最贴。
- **重点详读 + 其余快过**：先问用户这批里哪几篇是重点（和选题最贴近的核心文献、高被引/知名学者的代表作、用户自己读不懂要梳理的那几篇），重点篇出完整报告；其余的快速读一遍、出一行式要点（题目+一句话主张+类型+对本选题是否相关），不展开三问。

**默认是逐篇全读全出报告**——因为本来就要全读。重点详读那档只用在用户明确说“这批太多，重点几篇细读就行”时才走，别擅自只出几篇。

跟用户确认的是**选哪档进度安排**，不是减项：清单 25-30 篇太多也正常，本来就是大家都要读的。一批一批读，一篇一篇处理。

### 第二步：读文献

按第一步定的进度安排，一篇一篇读。MD 直接读；PDF 用 pdf skill（pdfplumber 提取）。读全文，不只读摘要——摘要常虚高，看不出论证。读一篇出一篇报告，不必等整批读完才输出。

### 第三步：逐篇出阅读报告

每篇按“三问骨架 + 精读清单”出一份报告。报告写到什么详略：**能让用户几个月后不翻原文也能想起来这篇讲了啥、怎么论证、对自己有什么用**。不必复述全文，抓骨架和启发。报告模板见下节。每篇一份 markdown，读一篇出一篇，不必等整批齐。

### 第四步：把报告给用户读，收他的想法落盘

报告是给用户读的，不是出了就完。预期流程：用户拿到报告 → 对着报告（最好也翻一眼原文）→ 读的过程中会冒出各种想法（“这里我不同意”“这个点正好能说我那个现象”“作者这句我想在引言里用”“这篇和 X 篇有点矛盾”）。**这些想法是后续写作的重要素材，不能丢。**

agent 做两件事：

1. 把报告交给用户、请他读完说想法；用户说一句记一句，代他落到盘上
2. 跟报告存在一起，新建“阅读思考/用户想法/”文件夹，每篇文献一份 `想法_文献名.md`。里面记用户原话或近意转述，并标清该想法是被报告里哪一点勾出来的（对应到报告的具体子项或子子需要点）。“我不一致意”“这里能用于引言”“记着查 Y 作者”“与 X 篇有点矛盾”这些都记、一概记。

重点：**不能代用户编想法**。用户想法是真实素材，agent 代笔会走偏。用户不说就空着，别凑数；用户说了就快记，尽重保留他的原话口吻。马理论科写作要的是用户自己的判断和语感，不是 agent 替他“技巧化”。

### 第五步：出一份“重点文献”小结

单篇报告之外，给一份一页纸的小结：这批里哪几篇是重点（值得回头写正文时重点复用、走 cowrite 时重点跟用户聊）、各自一句话点题。这份小结是给 marx-paper-cowrite 引导式用的“先读哪篇”依据，存在“阅读思考/”根目录。

### 第六步：跟用户过一遍

把报告、用户想法记录、重点小结一起给用户过一眼，让他看看有没有理解偏的地方——agent 对马理论科文献的解读可能有偏差（尤其对经典文本的义理判断），用户一句“这块理解反了”就能纠回来。**别闷头出完就走**。此步用户补充的想法也同步补进“用户想法/”。

## 阅读报告模板

每篇一份 markdown，按下面结构写。模板里【】是占位，按实际填。

```markdown
# 阅读报告 ·【文献题名】

- 作者：【】
- 来源：【期刊/出版社，年，期/卷】
- 类型：【理论阐释型 / 文本解读型 / 历史考辨型 / 比较研究型 / 对策研究型 / 实证调研型】
- 与本选题关系：【核心 / 相关 / 背景】

## 一、What —— 核心主张

- **核心问题**：作者要回答的问题是什么
- **核心观点**：一两句话，作者的主张
- **核心概念**：作者反复用、当分析工具的几个关键概念

## 二、How —— 如何论证

- **主要论证逻辑**：分几步、怎么把论点立起来（可列层级）
- **研究方法/路径**：文本解读 / 历史考辨 / 比较研究 / 理论分析 / 实证调研（如实标）
- **理论贡献**：验证/扩展/修正既有理论、提出新概念、建构新框架、整合零散说法、完善操作化（择实填）
- **精读要点**：
  - 有意思的地方在哪：
  - 是否已有主流解释，不足在哪：
  - 相比可替代解释，好在哪：
  - 模块精读（按本文结构择相关块写）：
    - 引言/文献综述：依托的核心理论/文本、对现有研究判断、如何避免罗列感、意义如何点明
    - 研究方法/路径：用了什么、为何适用
    - 主体论证：分几步、如何回应研究问题、如何让材料成证据
    - 讨论与结论：如何体现理论价值、未来方向

## 三、So what —— 对我有何启发

- **和已知知识的关联**：印证/推翻/补空白
- **对本选题的可用之处**：能借什么（论点/概念/角度/结构），哪里不能照搬
- **批判性思考**：我的保留意见、不同看法
- **是否重点文献**：是 / 否（理由）

## 四、学写作（可选，但建议写）

- 标题怎么起的
- 各级标题怎么排、各部分怎么衔接、起承转合怎么搭
- 我写自己论文能照搬什么结构手法
```

模板是骨架，不是填空表——某篇某节没东西就略掉，别硬凑。实证调研型文献把"研究方法/路径"那栏写实证细节；纯理论文献那一栏写"文本解读/理论分析无实证"即可。

## 衔接

- **上游**：marx-paper-topic 找来文献后进入；也可用户自己拎一批文献来读
- **下游**：报告服务于两件事——① 定结构（marx-paper-structure）：读过后用户对各文献骨架有底，搭结构更有数；② 引导式写作（marx-paper-cowrite）：本 skill 产出的"重点文献"小结告诉 cowrite 该重点跟用户聊哪几篇。整理引用素材（marx-paper-research）在定结构之后另做，不在本步
- **跳过**：急用可跳过本步直接 marx-paper-structure，不强迫

## 备注
