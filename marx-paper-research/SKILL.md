# 马克思主义理论学科论文素材整理

## 这个 skill 解决什么问题

选题定了、文献找来了（通常 25-30 篇，由 marx-paper-topic skill 提醒用户去找），结构也定好了（由 marx-paper-structure skill 搭好——整理间接引用时要对照结构判断有没有用，所以结构得先定），接下来要把这些文献里能用的东西提取出来。

文献里的引用分两种，整理方式不同：

- **直接引用**：带双引号的、标了具体出处的原文。不只是语录式的讲话原话，也包括著作、文章里的话。按来源（领导人/马恩/列宁/毛泽东/国外学者/政策文件等）分文件夹，全部整理。
- **间接引用**：没有双引号但标了脚注/尾注的转述。不全整理——对照结构判断每条有没有用，有用的才整理，按结构的各部分分文件夹，每篇参考论文最多取 1 条。

另外单独整理**数据素材**（统计报告、调研数据、年度报告等里的数据），连同来源一起提取。

三种素材都要连同**出处标注信息**一起提取，后面写论文时复用，不用再翻原书。

## 素材库的位置

素材库建在**工作区根目录**，与"参考文献"文件夹**同级**，不要放进参考文献里面。结构如下：

```
工作区/
├── 参考文献/          ← 用户下载的论文放这
├── 阅读思考/          ← marx-paper-read 建（可选）：阅读报告 + 用户想法，认知性保质
├── 素材库/            ← 本 skill 建，与参考文献同级，素材性保质
│   ├── 直接引用/
│   ├── 间接引用/
│   └── 数据/
├── 论文结构/
└── 论文正文/
```

注：若前面跑过 marx-paper-read，工作区里已有“阅读思考/”文件夹（阅读报告 + 用户读后想法）；本 skill 不动它。本 skill 建的“素材库/”是另一回事——装的是引用原文 + 出处，准备进脚注。

建之前征得用户同意，别擅自创建目录。

## 工作流程

### 第一步：读文献

用户找来的文献可能是 PDF 或 MD：
- **MD**：直接读，最省事。
- **PDF**：用 pdf skill 读（pdfplumber 提取文本）。PDF 也能读，只是慢一些。

**关于转 MD**：建议用户尽量把 PDF 转成 MD（更好读、省 token），但不强求。如果用户愿意转，**不建议让模型直接转**（耗 token 量大），推荐用 **MinerU** 软件——基本能覆盖大家的转 MD 需求。MinerU 转出来的形式是"每篇论文一个文件夹"（里面含 MD 文件和其他附件），建议在工作区建一个临时文件夹（如"待转MD_临时"）供用户把转好的文件夹暂存在那儿，之后再整理成参考文献的标准结构（每论文一个子文件夹 = 论文名_作者，里面一个 PDF + 一个 MD）。多余的东西先扔垃圾站（见下方"垃圾站规则"）。

### 第二步：跟用户确认直接引用的分类

直接引用按来源分文件夹整理。注意"原话"不只是讲话的语录式原话，也包括著作、文章里的话——只要是带双引号、标了具体出处的原文，都算直接引用。常见这几类来源：

- **领导人原话**（党和国家领导人的讲话、文章、著作里的原话）
- **马克思恩格斯原话**
- **列宁原话**
- **毛泽东原话**
- **国外学者原话**（非马克思主义的国外学者，如西方马克思主义各流派、其他国外人文社科学者）
- **政策文件**（中央文件、规划纲要、政策意见等里的原文）
- **其他学者原话**（国内学者论文里的根本性原话）

但具体情况具体判断——不是每篇文献都会出现所有这些来源，也可能有其他来源。所以整理前**先跟用户商量**：根据这批文献的实际情况，要分几类、每类叫什么，让用户确认。别擅自定死分类，也别漏掉用户实际需要的类别。

**顺便跟用户确认哪些是高被引/知名大佬**：marx-paper-topic 找文献时已按"关键词组/近期/高被引"分类放了文件夹。整理时再跟用户核对一眼：这批文献里哪几篇/哪几位作者是高被引或知名学者，后面间接引用二手论文核心观点时，被引学者必须是这类人。不引用无法核实/小众作者的细节观点。把高被引/知名者显式标记在素材库记彔里，后面写作时一眼能看出。

### 第三步：整理直接引用

用户确认分类后，在工作区根目录建"素材库/直接引用"文件夹，下面按分类建子文件夹。**建之前征得用户同意**。

每篇文献读完后，把里面的直接引用提取到对应分类。每条要记录两样东西：

1. **原话内容**本身
2. **出处标注信息**——这是关键，要提取的是**那个脚注指向的完整文献信息**，不是脚注编号：
   - 谁说的/写的（主要责任者）
   - 出自哪本书或哪篇文章（文献题名）
   - 哪个版本（出版社、出版年、译者等）
   - 哪一页（起止页码）
   - 也就是说，要提取到能直接写成 GB/T 7714 脚注的程度

**为什么出处信息要提取到这个程度**：后面 marx-paper-format skill 写脚注时需要"主要责任者.文献题名[M].出版地：出版者，出版年:起止页码"这种完整信息。如果只标"原文脚注[3]"，不知道脚注[3]对应的是哪本书哪一页，没法写规范脚注。所以要把脚注背后的具体文献信息提取出来，不是只记脚注号。

**直接引用的选取原则**：
- 领导人、马恩列毛等人物的原话：尽量收集，这些都是权威来源
- 国内学者的原话：只取其论文里的**根本性话语**（核心论点、关键判断），不取细节琐碎表述
- 直接引用可以多复用，这是常见做法
- **引号=原文（逐字）是硬规则**：直接引用提取到素材库时，引号里的内容必须与原文逐字一致。整理时已经要逐字核对。若只是意思接近但措辞不同的转述，不得标成直接引用上引号，应归到间接引用。后面写作复用时也不得在引号内增、删、改字。
- **经典原著可逐字的，优先归直接引用**：读文献时如果看到里面引用了马克思/恩格斯等经典原著的原话、能逐字核实的，应作为直接引用整理（标原书出处）；不要把原著里的原句降级成间接引用。间接引用的一般是二手论文/他人著作中对原著的转述与论断，不是原著本身的话。

### 第四步：整理间接引用

间接引用是参考论文里没有双引号但标了脚注/尾注的转述——读到小标（脚注/尾注标记）就知道前面那句是引用，再看有没有双引号判断是直接还是间接，没有双引号就是间接。

间接引用**不全整理**，由 agent 自己对照 marx-paper-structure skill 定好的结构判断每条有没有用（**判断由 agent 自己做，不要推给用户**——用户请 agent 来就是干这个的，不要跟用户说"我们一起过一遍"）：

1. 读到间接引用，先判断它对应结构的哪个部分、对自己论点有没有帮助
2. 有用的才整理：把参考论文里那段间接引用的原段截取下来，连同它标的出处信息一起记录（出处信息要提取到完整文献信息的程度，同直接引用）
3. 没用的跳过，不用全捞

整理出来的间接引用，按**结构的各个部分**分子文件夹存放（比如结构有三大标题，就分三个子文件夹；每个子文件夹里放对应该部分能用的间接引用）。这样写作时哪个部分需要间接引用支撑，直接去对应文件夹取。没归入任何部分的先放"待筛选"文件夹。

**每篇参考论文最多取 1 条间接引用**——避免过度依赖单篇文献，保证素材来源分散。25-30 篇文献各取最多 1 条，间接引用素材量也够用了。

**间接引用应指向二手论文的核心论断**：不要拿经典原著本身的句子去做间接引用。原著里能逐字引用的，应归直接引用上引号（传统马理论论文对经典原著是以直接引用为主）；间接引用一般是对二手论文/他人著作中对原著的转述与论断。原文已经说过的话，不要"读了原著不取原文，反而把它转述成间接引用"。

**间接引用的被引学者必须是高被引/知名者**：间接引用二手论文核心观点时，被引学者应为高被引或知名学者（第二步已跟用户确认过）。不引用无法核实/小众作者的细节观点。

### 第五步：整理数据素材

除了直接引用和间接引用，文献里还常有**数据**：统计报告数据（如 CNNIC 互联网报告）、调研数据、年度报告、百分比、样本量等。这些数据后面写引言、论证现状时常用。

在工作区根目录建"素材库/数据"文件夹，把文献里能用的数据连同**来源**一起提取：
- 数据本身（如"截至2024年12月我国网民规模达11.08亿人"）
- 来源（如"《第55次中国互联网络发展状况统计报告》"）
- 如果来源是报告/文件，标注清楚是哪个报告、哪一年、哪一页（能查到的话）

### 第六步：产出素材库

整理完，素材库结构如下：

```
素材库/
├── 直接引用/
│   ├── 领导人原话/
│   ├── 马克思恩格斯/
│   ├── 列宁/
│   ├── 毛泽东/
│   ├── 国外学者/
│   ├── 政策文件/
│   └── 其他学者原话/
├── 间接引用/
│   ├── 一、xxx/
│   ├── 二、xxx/
│   ├── 三、xxx/
│   └── 待筛选/
└── 数据/
```

每个文件夹里的素材，每条都带完整出处信息。这个素材库就是写作时的"弹药库"。

## 垃圾站规则（通用）

整理过程中要删除多余文件时（如重复的 PDF、转 MD 的中间产物、没用的附件等），**不要直接删**——先在工作区根目录建一个"垃圾站"文件夹，把要删的东西扔进去。用户明确要求删除时才真删。这样万一删错了还能捞回来。

这条规则适用于整个系列的所有 skill——任何时候要删东西，都先扔垃圾站。

## 衔接

整理完素材库后，衔接写正文的 skill（**marx-paper-quickwrite** 快速式或 **marx-paper-cowrite** 引导式）搬写正文：直接引用直接用、出处信息复用（人物原话要溯源到原始出处，见写作 skill 的溯源规则），间接引用换说法用、每篇最多 1 条、少复用层层转述的间接观点。写作完成后再衔接 **marx-paper-format**（格式）skill：format skill 会把出处信息编排成规范的脚注和参考文献表。其中**复用整理素材里原话的引用**，format skill 不必再去原 PDF 核实页码（别人已核实过），但会在完成后于对话里告知用户哪些是复用的，让用户心里有数。

**关于期刊论文著录信息**：若素材涉及期刊论文的完整著录（刊名、年、卷期、页码），提醒用户去知网该论文页面，点那个“引用”图标按钮复制标准著录信息回来贴上，不要让写作/format 凭空凑。这条著录信息准不准，直接关系到参考文献表合不规范。

**关于经典原著的版本**：整理原话出处时，留意该经典是否有更新/更全的版本。复用别人标的老版本页码不能直接拿，能够查到最新版本页码的应查，查不到或不确定的标出原始版本号、提醒用户核实。系列后续会做一个独立的原著页码检索库 skill 来集中确认。
