# 学术论文格式规范与脚注参考文献

## 这个 skill 做什么

把中文学术论文草稿（markdown 或 docx）加工成格式规范、可直接提交的 Word 文档。覆盖人文社科类论文的全流程格式：引用标注（脚注+参考文献）、页码核实、Word 排版（字体、字号、行距、缩进、分页、颜色）。

核心规则（这几条是国标，不可商量）：
- **脚注**：页下注，每页重新编号，圈码 ①②③，凡引用必标注，必须标具体页码
- **参考文献**：只列正文脚注中出现过的文献，不标页码，不重复列，未引用的不得出现

## 工作流

### 第一步：摸清项目环境

打开 skill 后第一件事不是动手加脚注，而是先看清所处项目的全貌。这一点很重要——不同论文项目的目录结构、源文献位置都不一样，盲目动手会找不到 PDF 也搞错格式要求。

具体做这些：
- 扫描当前工作目录及其上层，找到论文草稿（markdown 或 docx）
- 找可用的源文献：原书 PDF、原刊 PDF、电子书等
- 找格式模板或撰写要求文档（如学校/学院的论文规范）
- 然后跟用户确认三件事：
  1. 源 PDF 在哪个目录？（有些项目可能根本没有原 PDF，这点要明确问）
  2. 这次正文行距多少？（毕业论文常为多倍行距 1.25，课程论文常为 1.5 倍行距——以用户本次作业要求为准）
  3. 有没有指定的字体/格式模板？

不要假设源 PDF 一定存在。没有 PDF 就如实告诉用户，走"页码待确认"流程，绝不能编。

### 第二步：通读草稿，定位所有需要脚注的位置

逐段通读全文，找出每一处：
- **直接引用**：引用了原文原话（哪怕几个字）
- **间接引用**：概括、转述了他人观点、理论、论断

这一步要尽量找全。GB/T 7714 的要求是"凡引用必标注"——漏标在学术规范上等同于学术不端。宁可多标，不可漏标。把每一处需要脚注的位置和它引用的是哪本文献记下来。

### 第三步：核实页码（最关键的一步）

这是最容易偷懒、也最不能偷懒的一步。脚注里的页码必须是引文在原文献中的真实页码。

- **有源 PDF**：用 pdf skill（pdfplumber/pypdf 提取文本，Poppler 渲染页面为图片做视觉核对）定位引文所在页，记录真实页码。
- **无源 PDF**：明确告诉用户"这条引用的页码我无法核实"，请用户提供，或在脚注里标注"[页码待确认]"等用户补。
- **绝不能凭感觉编页码。** 这是学术诚信的底线。

为什么必须读 PDF 而不是 markdown：同一文献的 markdown 电子版和 PDF 版页码往往不一致——markdown 没有固定的"页"概念，分页随渲染方式变化。PDF 才是引文页码的权威依据。

### 第四步：编写脚注

按 GB/T 7714-2015 格式编写每条脚注。完整格式见 `references/gbt7714-format.md`，这里列最常用的：

- **专著**：`主要责任者.文献题名[M].出版地：出版者，出版年.起止页码.`
  - 例：`卢卡奇. 历史与阶级意识[M]. 北京: 商务印书馆, 1999: 143-149.`
- **期刊**：`主要责任者.文献题名[J].刊名，年，卷(期)：起止页码.`
- **译著**：`原著者.文献题名[M].译者，译.出版地：出版者，出版年.起止页码.`

几个易错点（务必注意）：
- 出版地后用**全角冒号**"："，不是英文冒号
- 页码直接写数字范围，**不加"第"和"页"字**（如 `143-149`，不是"第143-149页"）
- **重复引用**：连续引用同一文献用"同上（书），第╳页"（注意这里反而保留"第"和"页"）；非连续引用只注著者、文献名、页码
- 编著/译著在编者、译者姓名后加"编""译"

### 第五步：生成参考文献表

在文末生成参考文献，规则严格：
- **只收录**正文脚注中出现过的文献
- **不标页码**
- **不重复**（同一文献多处引用，参考文献表里只列一次）
- 按文中**首次引用顺序**编号 [1][2][3]...
- **未引用的文献不得出现**——这条是硬性规定。把只是"思想上参考过"但正文没有实际引用的文献列进去，属于违规

为什么这条这么重要：GB/T 7714 和各高校规范都明确写"未引用的文献不得出现"。参考文献表是"被引用文献的完整列表"，不是"我读过的书单"。这个区别要刻在脑子里。

### 第六步：渲染成 Word

用 pandoc 把 markdown 转成 docx，再用 `scripts/render_docx.py` 后处理格式：

```bash
pandoc 草稿.md -o 临时.docx --from=markdown --to=docx
python scripts/render_docx.py 临时.docx 成品.docx --line-spacing 1.5
```

后处理脚本负责：
- 脚注编号：设置为每页重新编号（OOXML 的 numRestart=perPage）
- 脚注编号格式：尝试设为圈码 ①②③（见下方"已知技术事项"）
- 标题：黑体，居中
- 小标题：黑体
- 正文：宋体小四（12pt），首行缩进 2 字符
- 参考文献：宋体/Times New Roman 五号（10.5pt），序号左顶格 [1][2]
- 行距：按用户指定（默认 1.5）
- 页边距：上下 2.54cm，左右 3.18cm

### 第七步：自检

交付前逐项检查：
- [ ] 脚注数量与正文引用标记一致
- [ ] 参考文献每条都能在脚注中找到对应（无多余、无遗漏）
- [ ] 所有页码已核实，无"[页码待确认]"残留（或已明确告知用户哪些待确认）
- [ ] 脚注格式：圈码、每页重编号、全角冒号、页码无"第/页"字
- [ ] 字体/行距/缩进正确
- [ ] 打开 docx 目视检查一遍脚注是否真的出现在页面底部

## 修补已有论文

当输入是已经写好的论文（可能有也可能没有脚注）时，工作流调整为：
1. **扫描现有脚注**：逐条核实页码和格式是否正确，错误的改正
2. **找漏标**：找出"该加脚注但没加"的位置，补全
3. **重建参考文献表**：基于所有脚注重新生成，确保配套
4. **重新渲染**：用脚本输出格式正确的 docx

## 已知技术事项（迭代时关注）

**圈码脚注 ①②③——已解决**：用 `customMarkFollows="true"` 方案。在正文每个 `w:footnoteReference` 上加该属性，并在 run 里写入圈码字符 `<w:t>①</w:t>`，Word 就显示我们指定的字符而非自动编号。底部脚注的 `w:footnoteRef` 同步替换成圈码字符（不加 FootnoteReference 样式，避免上标变小）。`w:id` 关联保留，仍是真脚注（删正文标记，底部脚注跟随消失）。由 `scripts/footnote_circle_marks.py` 实现。

**每页重新编号——已解决**：用 LibreOffice headless 把 docx 转 PDF，再用 pdfplumber 扫描每个圈码出现在哪一页，按页分组后每页内按出现顺序重新赋 ①②③。由 `scripts/renumber_footnotes_per_page.py` 实现。依赖 LibreOffice 和 pdfplumber。注意：编号是渲染时算死的，不是实时自动的——用户在 Word 里手动新增脚注后需重跑脚本。这与本项目工作流（markdown 改 → 脚本渲染 → 交最终版）匹配。

**pandoc 脚注 ID 偏移**：pandoc 从 markdown 的 YAML frontmatter 会预留一些内部脚注 ID，导致正文脚注的 XML ID 不是从 1 开始（如从 9 开始）。这不影响圈码显示（圈码是我们按出现顺序算的，与 XML ID 无关），但开发调试时要心里有数。

## 相关文件

- `scripts/render_docx.py` — pandoc 转换 + docx 后处理脚本，负责字体、行距、脚注编号格式
